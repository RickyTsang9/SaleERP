# SaleERP 管理系统更新日志

## 2026-04-14

### 本次版本重点

- 完成经营驾驶舱、预算计划、经营简报、经营决议事项、经营快照等管理层能力落地。
- 完成销售、采购、仓储、财务主流程的名称化展示、状态中文化、联动跳转和待办化优化。
- 完成多项高风险操作漏洞修复，补齐后端状态校验、主数据删除保护、重复提交与重复作废拦截。
- 完成前端高频页面运行时问题修复，包括菜单图标、品牌名称、消息中心、经营看板兼容模式、时间格式化缺失等问题。
- 完成 GitHub 发布仓库同步，整理前后端最新代码与数据库脚本。

### 新增功能

- 新增管理驾驶舱、经营健康度、风险雷达、经营预警待办。
- 新增年度预算计划、预算审批、预算版本管理。
- 新增经营简报归档、经营决议事项台账、经营快照归档。
- 新增高风险客户、重点付款供应商、资金压力窗口、经营目标节奏分析。
- 新增采购订单与采购入库、应付台账、付款流水的来源联动。

### 业务体验优化

- 销售订单、采购订单、采购入库、销售出库、库存台账、库存流水、盘点单、调拨单统一优化为名称优先展示。
- 客户、供应商、仓库、商品、库位等主数据下拉改为远程搜索。
- 首页、经营看板、消息中心、待办入口支持路由参数承接和一键跳转处理。
- 财务链路收口为来源单据驱动，回款、付款与台账联动更清晰。
- 客户跟进、数据备份等页面增加待办化和风险提示。

### 缺陷修复

- 修复经营看板在兼容模式下数据被异常清空的问题。
- 修复部分页面缺少 `parseTime` 导入导致的运行时报错。
- 修复菜单缺失图标、系统名称残留旧品牌、公告消息展示异常等问题。
- 修复主数据被删除后业务页面直接暴露历史内部 ID 的问题，统一改为友好提示。
- 修复后端接口可绕过前端限制修改已流转单据、删除已引用主数据等安全问题。
